# c 프 팀플 구조설계서

## Overworld + Field 분리형 CLI 탐험 게임 구조

## 1. 설계 목표

#### 본 시스템은 C 언어 기반 CLI 탐험 게임으로,

여러 개의 필드(Map)를 연결한 Overworld **기반 구조** 를 사용한다.
이번 개정의 핵심 목표:
단일 맵 구조 → Field + Overworld **구조로 확장**
필드 데이터를 코드 배열로 직접 설계 가능
필드 배치와 데이터 완전 분리
확장성 및 유지보수성 극대화

## 2. 전체 구조 개요

```
Input
→ Game→ Overworld
→ Field(Map)
→ Systems→ Entity
→ Render→ Log
```
## 3. 핵심 구조 개념

## 3. 1 Field ( 기존 Map 개념 )

### 역할

#### 게임 화면 하나 (단일 필드)

#### 타일 데이터 및 충돌 처리


### 특징

```
필드마다 크기(width, height) 다름
2 차원 배열 기반
```
## 3. 2 Overworld ( 기존 Stage 개념 )

### 역할

```
여러 Field를 연결하는 구조
현재 필드 위치 관리
필드 간 이동 처리
```
## 3. 3 Field 데이터 분리

```
field1.c, field2.c 등은 순수 데이터 파일
필드 배열만 포함
로직 없음
```
